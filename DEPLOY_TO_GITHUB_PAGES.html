无需Git命令的GitHub Pages部署指南

由于您的系统可能尚未安装Git，这里提供一种更简单的方法来部署您的统计Web应用到GitHub Pages。

步骤1：构建项目

项目已经构建完成，生成的静态文件位于 dist 文件夹中。

如果您需要重新构建，可以双击运行项目根目录下的 build.bat 文件（如果存在），或者按照以下步骤操作：

  1. 打开命令提示符(cmd)或PowerShell
  2. 导航到项目目录：cd C:\Users\gk\Documents\Trea CN\statistical-webapp
  3. 运行命令：npm run build

步骤2：创建GitHub仓库

  1. 访问 GitHub.com 并登录您的账号
  2. 点击右上角的 "+" 按钮，选择 "New repository"
  3. 输入仓库名称：statistical-webapp
  4. 选择公开或私有仓库
  5. 不要勾选 "Initialize this repository with a README" 选项
  6. 点击 "Create repository" 按钮

步骤3：上传文件（无需Git）

GitHub提供了直接上传文件的功能，无需使用Git命令：

  1. 在新创建的仓库页面，点击 "Upload files" 按钮
  2. 在上传界面，您可以直接拖动 dist 文件夹中的所有文件到浏览器中
  3. 或者点击 "choose your files" 按钮，浏览到 C:\Users\gk\Documents\Trea CN\statistical-webapp\dist 文件夹，选择所有文件
  4. 滚动到底部，在 "Commit changes" 部分输入提交信息，例如 "首次部署"
  5. 点击 "Commit changes" 按钮上传文件

步骤4：配置GitHub Pages

  1. 上传完成后，返回仓库的主页面
  2. 点击顶部的 "Settings" 选项卡
  3. 在左侧导航栏中，点击 "Pages"
  4. 在 "Source" 部分，从下拉菜单中选择 "main" 或 "master" 分支
  5. 确保选择的文件夹是 "/ (root)"
  6. 点击 "Save" 按钮
  7. 等待几分钟，页面会显示您的GitHub Pages网站URL

部署完成！

您的统计Web应用已经成功部署到GitHub Pages。您可以通过显示的URL访问您的网站。

注意：GitHub Pages更新可能需要几分钟时间才能生效。

提示

如果您将来想更新网站，只需重新构建项目，然后按照步骤3中的方法上传新的文件即可。

如果您想学习使用Git命令进行部署，建议您安装Git后再尝试。
